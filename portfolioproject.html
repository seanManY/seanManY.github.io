HCI Portfolio Project

This was the final project for my class User Interaction Software. The project was to create a personal portfolio website utilizing human computer interaction programming. Tasks were broken down into 4 sections, User Research, Implementation (prototyping), User Testing, Implementation (Iterations.

User Research

In this step I was to research and plan out the content to be included in the website. This was broken down into multiple steps whcih included, Compettitor Analysis, User Interviews, Feature Value Matrix and Feature Prioritation Diagram.


Competitor Analysis

I wanted to create a porotfolio to showcase my game projects, therefore I researched other game developer's protfolios. From my research I found that most develpoers prioritize three pages.

  • Project Page - A page featuring their work
  • About Page - A page deescribing who they are and their specialties
  • Contact Page - A page to get into contact with them, either via contact form or general contact information (email, phone)

User Interviews

The User interviews were conducted in order to get a wider perspective user needs in other fields. I intervied people in non-game and non-tech fields to see what information they value in their own portfolio. This data was utilized in order to form my feature list.


Feature Value Matrix/ Diagram

Once comparing other portfolios I came up with a feature list for my website. I then took that feature list and rated them form 1-5 in various categories (User Value, Business Value, Development Effort). These values were then totaled up and ranked by priority with the higher number being the higher priority. To further visualize the feature list, the features were put into a graph to show which feature I should prioritise first.


Implementation (prototype)

The next step was to create a website. The first step in this process was to sketch out the layout of site. I sketched out wireframes and designed what content would go where. The challenge with step was making the site responsive in order to be viewed on mobile devices. I implemented the responsiveness myself in addition to creating my own grid system via CSS, in order to fully understand the implementation.





User Testing

After impementation I was tasked to do user testing, where I conducted surveys and congnitive walkthroughs in order to gather information. I received two major points of feedback. First was to add color, originally my prototype lacked color and had a minimal tone. The other peice of feedback was to show users my games by allowing users to play my projects.


Implementation (Iteration)

I gathered the feedback from user testing and used that information to update my website. I added my color to my website and making it more playful. I also added adtional links on the page that redirects the user to web-based versions of my games for them to play.
